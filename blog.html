Question no-1:Discuss the scope of var, let, and const

var: var has functional-level scope and can be redeclared and updated throughout the function. var declarations are hoisted to the top of their containing function or global scope.

let: let has block-level scope and allows reassignment but not redeclartion in the same scope.let are also hoisted, but they are not initialized

const:const also has block-level scope and is used for variables that should not be reassigned after initial declartion. As well as const are also hoisted like let and var , but they are not initialized

Question no-2:Tell the use cases of null and undefined

undefined: undefined means that the value hasn't been set or doesn't exist yet. It's like an empty slot waiting for a value.

null: This is like an intentional empty slot. It's used when you want to say, "I want this to be empty on purpose." It's a way to tell the code that there's no value there.

Question no-3:What do you mean by REST API?

REST API: An API, or application programming interface, is a set of rules that define how applications or devices can connect to and communicate with each other. A REST API is an API that conforms to the design principles of the REST, or representational state transfer architectural style.
